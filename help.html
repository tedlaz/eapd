eΑΠΔ — Οδηγός Χρήσης

1) Σειρά Εγγραφών

  • Γενικά Στοιχεία: Εμφανίζεται μόνο μία φορά, στην κορυφή.
  • Στοιχεία Ασφαλισμένου: Εμφανίζεται μία φορά για κάθε εργαζόμενο.
  • Μέσα σε κάθε ασφαλισμένο Στοιχεία Ασφάλισης: Εμφανίζεται μία τουλάχιστον φορά και μπορεί και περισσότερες για κάθε τύπο αποδοχών.
  • Μέσα σε κάθε ασφάλιση Στοιχεία Επιδότησης: Εμφανίζεται μόνο όταν υπάρχει επιδότηση μια φορά για κάθε τύπο επιδότησης με προυπόθεση να έχουμε πλήρες ωράριο και όλες εργάσιμες.
  • Τέλος Αρχείου: Εμφανίζεται μόνο μία φορά, στο τέλος.

2) Κύριοι Κανόνες Υπολογισμών

  • No46 = No45 * emp / 100 (από kpk.js)
  • No48 = No45 * tot / 100 (από kpk.js)
  • No47 = No48 - No46
  • No51 = No48 - No52 - No53
  • No65 = No62 + No64

Όλες οι αριθμητικές πράξεις στρογγυλοποιούνται σε 2 δεκαδικά.

3) Κανόνες Επιδότησης (epidotiseis.js)

  • No60: επιλογή από code + per
  • No61: pemployee
  • No62: vemployee ή αν είναι * τότε No45*pemployee/100
  • No63: pcompany
  • No64: vcompany ή αν είναι * τότε No45*pcompany/100
  • No65: No62 + No64

4) Κανόνες Επικύρωσης

  • No40/No41 κενά μόνο αν No42 ∈ {002, 004, 005, 006}
  • No56=0 ⇒ No44=0, No56=1 ⇒ No44>0
  • No54/No55 προαιρετικά (κενό ⇒ __)
  • No58 προαιρετικό (κενό ⇒ 00)
  • Έλεγχος μήκους πεδίων στο Recalculate totals

5) Σύνολα Header

  • No16 = άθροισμα No43 (όλων των StoixeiaAsfalisis)
  • No17 = άθροισμα No45
  • No18 = άθροισμα No51 μόνο

6) Αναζήτηση Κωδικών

  • No31: KAD lookup (osyk_kad.js)
  • No35: EID lookup (osyk_eid.js)
  • Η αναζήτηση είναι accent-insensitive (π.χ. ά = α)

7) Ημερομηνίες & Κείμενα

  • Ημερομηνίες εισαγωγής: dd/mm/yyyy
  • Κενή ημερομηνία στην εξαγωγή: 00000000
  • Στο Recalculate τα ελληνικά κείμενα γίνονται ΚΕΦΑΛΑΙΑ χωρίς τόνους

8) Ροή Εργασίας (προτεινόμενη)

  1. Συμπλήρωση δεδομένων
  2. Πάτημα Recalculate totals και διόρθωση πιθανών errors
  3. Πάτημα Generate APD text
  4. Αποθήκευση αρχείου με το σωστό encoding
